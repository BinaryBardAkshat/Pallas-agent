# Zettelkasten Note-Taking

## Description
Create atomic, interlinked notes in the Zettelkasten method. Each note captures one idea, linked to related notes to build a knowledge graph over time.

## When to Use
- Processing articles, books, or research into lasting knowledge
- Building a personal knowledge base
- Connecting ideas across domains
- Preparing for writing by building a network of related ideas

## Instructions
**Core principles of Zettelkasten**:
1. **One idea per note** — atomic. If you're writing two things, make two notes.
2. **Write in your own words** — no copy-paste. Reformulating = understanding.
3. **Always link** — every note must connect to at least one other note.
4. **Permanent notes** — write as if for a stranger who has no context.

**Note structure**:
```markdown
# [Unique ID] — [Title]
Date: YYYY-MM-DD
Tags: #tag1 #tag2

[Main content — 3-10 sentences in your own words]

## Links
- [[Related Note ID]] — why it's related
- [[Another Note ID]] — the connection

## Source
[Where this came from]
```

**When creating notes from content**:
1. Read/process the source material
2. Identify 3-7 atomic ideas worth keeping
3. Write one note per idea, in your own words
4. Link each note to existing notes in the vault
5. Create an index note if this starts a new topic cluster

## Examples
- "Create Zettelkasten notes from this article"
- "Add this book's key ideas to my Zettelkasten"
- "Link these notes about machine learning together"
